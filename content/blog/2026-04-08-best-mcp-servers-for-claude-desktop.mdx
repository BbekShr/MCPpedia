## What Scores Really Mean

On MCPpedia, scores (out of 100) factor in:

- **CVE history**: Known vulnerabilities
- **Code quality**: Well-written, maintainable code
- **Permission design**: Does it ask for more access than it needs?
- **Documentation**: Can you actually use it without guessing?
- **Maintenance**: Is the author responding to issues?

A score of 90+ means the server is production-ready. Anything below 75 is a yellow flag.
